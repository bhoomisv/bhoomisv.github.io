Cloud Migration

To modernize the data infrastructure by migrating code and data pipelines from the Hadoop ecosystem (HiveSQL, SparkSQL, ScalaSpark) to Google BigQuery (BQSQL). This transformation enables faster, more scalable, and cost-effective data processing, aligned with the company's cloud-first strategy.

Business goals:

Cloud Modernization, Operational Efficiency, Data Accuracy and Governance, Al-Driven Acceleration.

Workflow:

Client transfers source data from Hadoop (HDFS) to BQ table.

Using the Generative Al tool to convert HiveSQL/SparkSQL/Scala code to optimized BigQuery SQL (BQSQL).

Converted BQSQL code is deployed to Dev or Production (DALM machine) environments for execution.

Use the custom validation tool to compare Hadoop vs. BigQuery outputs (row counts, schema, numeric aggregations).

Display validation results through Looker dashboards for review by QA or business teams.
